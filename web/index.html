Agent loop

The settlement path is intentionally simple to inspect: perceive, pay, reason, settle, prove.

01 / PERCEIVE Ask for outcome

The agent targets a market near resolution and requests the verified result.

02 / PAY Handle 402

The endpoint requires payment; the agent signs and resubmits an x402-style header.

03 / REASON Validate receipt

The agent checks market id, winning side, and proof reference format.

04 / SETTLE Call Casper

The resolver calls submit_resolution and settle.

05 / PROVE Publish trail

The receipt id is stored as proof_ref and shown in this viewer.
