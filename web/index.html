Agent Flow

01 / PERCEIVE

The agent targets a market near resolution and asks the truth endpoint for the result.

02 / PAY

The endpoint returns HTTP 402; the agent resubmits with an x402-style signed payment header.

03 / REASON

The endpoint returns the verified winning side and receipt id.

04 / SETTLE

The agent calls submit_resolution and settle on Casper.

05 / PROVE

The receipt id is stored as proof_ref and re-emitted in settlement events.
